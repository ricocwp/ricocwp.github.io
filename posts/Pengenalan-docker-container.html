Pengenalan Docker Container

Apa sih Docker itu?

Docker adalah sebuah project open-source yang menyediakan platform terbuka untuk developer maupun sysadmin atau saat ini ada devops yang digunakan untuk membangun, mengemas, dan menjalankan aplikasi atau service dalam sebuah container. Docker dilengkapi dengan fitur sandbox yang menjamin pengerjaan oleh pengembang dan pihak lain tidak terganggu. Sandbox adalah istilah keamanan komputer dengan mekanisme pemisahan aplikasi atau program tanpa menganggu host lain. Oleh sebab itu, setiap container dalam Docker tidak saling terhubung atau terisolasi.

Sejarah.

Docker, inc. didirikan oleh Solomon Hykes dan Sebastien Pahl selama grup inkubator startup Y Combinator Summer 2010 dan diluncurkan pada 2011. Hykes memulai proyek Docker di Prancis sebagai proyek internal dalam dotCloud , sebuah perusahaan platform-as-a-service. Kontributor awal adalah insinyur dotCloud lainnya, termasuk Andrea Luzzardi dan FranÃ§ois-Xavier Bourlet. Jeff Lindsay juga terlibat sebagai kolaborator independen. Docker memulai debutnya untuk umum di Santa Clara di PyCon pada 2013. Ia dirilis sebagai open-source pada Maret 2013. Pada saat itu, ia menggunakan LXC sebagai lingkungan eksekusi default. Satu tahun kemudian, dengan rilis versi 0.9, Docker menggantikan LXC dengan komponennya sendiri, yang ditulis dalam bahasa pemrograman Go.

Docker Workflow

Arsitektur pada docker menggunakan mekanisme client dan server. Dimana docker client akan mengirim request ke docker daemon untuk membangun, mendistribusikan, dan menjalankan container. Docker daemon dan docker client dapat berjalan pada sistem yang sama dan berkomunikasi via socket menggunakan RESTful API.

Komponen Penting Dalam Docker

Docker memiliki beberapa komponen penting dalam menjalankan mekanisme kerjanya. Masing-masing komponen pada docker saling membutuhkan satu dengan yang lain, sehingga jika ada salah satu komponen yang tidak tersedia maka kinerja docker akan terganggu. Berikut beberapa komponen penting pada docker:

  1. Docker Image

    2. Docker image adalah komponen dasar dari docker container. Docker image ini berupa sebuah sistem operasi atau aplikasi yang telah diinstall dan dikemas menjadi sebuah file siap pakai. Image yang mendukung docker container ada banyak sekali, seperti nginx, redis, dll. Untuk info docker image yang tersedia, bisa mengunjungi Docker Hub

  2. Container

    3. Container merupakan sebuah tempat/wadah dimana docker image dikemas, container berjalan diatas image. Kita dapat melakukan instalasi aplikasi maupun sistem operasi didalam container, keunggulan container ini yaitu isolasi, jadi antara container satu container dengan container lainnya saling terpisah sehingga tidak menganggu proses lain sehingga mempermudah pekerjaan developer maupun devops.

  3. Docker Registry

    4. Docker registry adalah repository distribusi yang berisi kumpulan docker image terpusat baik bersifat public maupun private repository. Registry docker public repository bisa kita lihat di Docker Hub Registry dan pada docker registry ini, kita dapat melakukan pull maupun push image kita sendiri.

  4. Dockerfile

    5. Dockerfile merupakan sebuah file yang berisi skrip serangkaian perintah untuk melakukan otomasi dalam membangun image sesuai kebutuhan pada docker.

Perbedaan Docker Container dengan Virtual Machine

Secara fungsi, Virtual Machine (VM) dengan Container sama yaitu sebagai teknologi virtualisasi, tetapi ada beberapa perbedaan antara VM dengan Container antara lain:

  • VM menggunakan keseluruhan resource yang telah kita sediakan, sehingga jumlah resource yang kita pakai akan besar. Docker Container hanya menjalankan aplikasi tertentu yang kita inginkan saja, sehingga penggunaan resourcenya lebih sedikit daripada VM.

  • VM cenderung lebih berat daripada docker container karena menggunkan seluruh sumber daya yang disediakan. Docker Container menggunakan sumber daya lebih sedikit daripada VM sehingga lebih ringan.

Keuntungan Menggunakan Docker

  • Isolasi Pada Container

    • Aplikasi atau OS yang diinstal dalam container saling terisolasi, sehingga tidak menganggu antara container satu dengan container lainnya.

  • Kemudahan Konfigurasi

    • Dengan adanya container pada docker, kita dapat membuat atau menjalankan aplikasi atau OS secara mudah tanpa konfigurasi yang panjang dan tentunya meghemat banyak waktu.

  • Multi Cloud Platform

    • Salah satu poin utama dari docker adalah didukung oleh berbagai Cloud Platform seperti Google Cloud Platform (GCP) dan Amazon Web Service (AWS) sehingga dapat diintegrasikan dengan baik. Docker juga dapat terintegrasi dengan baik ada teknologi Infrastructure as a Service (IaaS) seperti OpenStack dan Microsoft Azure. Docker juga mendukung manajemen konfigurasi (otomasi) seperti Ansible dan Puppet.
