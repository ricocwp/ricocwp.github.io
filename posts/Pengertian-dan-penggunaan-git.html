Pengertian dan Penggunaan GIT

Git adalah sebuah software lama yang masih digunakan hingga saat ini.

Git sering digunakan oleh programer atau para kontributor pada proyek yang melibatkan kolaborasi.

Pengertian Git

Sebelum membahas mengenai git, kita akan bahas dahulu mengenai Version Control System (VCS) sebagai pondasi awal konsep git.

VCS (Version Control System) disebut juga revision control system atau source code management adalah sistem yang dapat mengelola perubahan dari sebuah dokumen, program komputer, website dan kumpulan informasi lain. Atau secara garis besar VCS ini adalah sebuah sistem yang dapat mengelola perubahan didalam suatu dokumen.

Kenapa kita menggunakan VCS?

  1. Sebuah sistem yang menyimpan 'rekaman / snapshot' perubahan pada source code
  2. Memungkinkan bekerja berkolaborasi dengan lebih baik dan efisien
  3. Mengetahui siapa yang melakukan dan kapan sebuah perubahan terjadi
  4. Memungkinkan kita untuk kembalai ke keadaan sebelum perubahan terjadi (checkout)

Git adalah sebuah VCS (software) terdestribusi untuk mengelola perubahan file di dalam folder (repository/repo). Git akan membuat history perubahan setiap file dengan menggunakan commit yang dibuat, lalu pada setiap commit tersebut, git akan membuat rekaman/snapshot terhadap perubahan file tersebut.

Ilustrasi Tanpa Menggunakan Git

Contoh kasus dalam pembuatan website portfolio sederhana. Dalam satu folder terlihat ada beberapa file. Saat ada beberapa perubahan pada file tersebut walau mungkin hanya perubahan kecil, maka akan dibuat file baru dan kita sulit mengetahui perubahan apa yang dilakukan, intinya kurang efisien dalam pengerjaan suatu proyek. Apalagi kalau proyek tersebut melibatkan kolaborasi dalam pengerjaannya, pastinya setiap kontributor memiliki source code masing-masing sehingga akan sulit untuk melakukan penggabungan source code menjadi satu website yang utuh.

Ilustrasi Dengan Menggunakan Git

Contoh kasus masih tetap yaitu pembuatan website portfolio sederhana. Dengan menggunakan git terlihat lebih efesien. Dalam satu folder tersebut sekarang hanya terdapat satu file Portfolio.html tetapi kita bisa tau pernah melakukan perubahan apa saja dalam file tersebut. Setelah kita menginisialisasi folder tersebuat dengan git, maka git akan menganggap folder tersebut sebagai repository/repo. Git akan mengawasi setiap perubahan yang terjadi pada saat kita melakukan commit pada perubahan tersebut, setelah kita melakukan commit saat ada perubahan, maka git akan membuat snapshot/rekaman secara otomatis. Fungsi snapshot ini dapat digunakan untuk mengulang perubahan yang sudah terjadi sesuai keadaan snapshot yang kita inginkan (pada kode snapshot). Dalam satu commit berisi informasi penting seperti siapa yang melakukan commit, waktu terjadinya commit, dan pesan perubahan apa pada commit tersebut. Dengan begitu, penggunaan git sangat membantu dalam penyelesaian suatu proyek, apalagi proyek kolaborasi karena lebih terorganisir dan efisien dalam pengerjaannya.

Penggunaan git saat ini masih sangat populer di kalangan programer dan penggiat proyek open source, karena pada dasarnya git sendiri dibuat oleh Linus Trovalds (Pencipta Linux) untuk tujuan pengembangan proyek open source sehingga kita dapat berkontribusi dimanapun, kapanpun di seluruh penjuru dunia sesuai pedoman komunitas yang berlaku.
